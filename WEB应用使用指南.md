# 教案合规检测 Web 应用使用指南

## 🚀 快速开始

### 1. 安装依赖

首先确保已安装所有依赖包：

```bash
pip install -r requirements.txt
```

新增的依赖：
- `flask>=2.3.0` - Web框架
- `flask-cors>=4.0.0` - 跨域支持

### 2. 配置环境变量

确保 `.env` 文件已正确配置：

```env
DEEPSEEK_API_KEY=your_deepseek_api_key
DEEPSEEK_BASE_URL=https://api.deepseek.com
QWEN_VL_BASE_URL=http://xn-d.suanjiayun.com:57915/v1
QWEN_VL_MODEL=Qwen/Qwen3-VL-235B-A22B-Instruct-FP8
```

### 3. 启动Web服务器

```bash
python web_app.py
```

启动成功后，你会看到：

```
======================================================================
教案合规检测 Web 应用
======================================================================

✅ 检测器初始化成功

🌐 访问地址: http://localhost:5000
📝 请在浏览器中打开上述地址使用Web界面

按 Ctrl+C 停止服务器
======================================================================
```

### 4. 打开浏览器

在浏览器中访问：**http://localhost:5000**

## 📖 界面功能说明

### 左侧栏：文档选择区

#### 1. 选择文档
- 点击 **"选择Markdown文档"** 区域
- 选择本地的 `.md` 文件
- 系统会自动读取并显示文件内容

#### 2. 文件信息
选择文件后，会显示：
- **文件名**
- **文件大小**
- **文件路径**

#### 3. 最近使用
- 显示最近使用的10个文档
- 点击可快速重新选择

#### 4. 检测规则
- 显示当前系统包含的规则数量（19条）
- 点击 **"查看规则详情"** 可查看所有规则

---

### 中间栏：文档预览区

#### 1. 文档渲染
- 保留Markdown原始格式
- 自动渲染标题、段落、表格、图片等
- 移除页码标签（`<page N>`）用于显示

#### 2. 违规高亮
- 检测后，违规内容会以**红色背景**高亮显示
- 鼠标悬浮可查看**违规类型**
- 点击高亮区域可跳转到右侧对应的违规项

#### 3. 工具栏
- **切换高亮**：开启/关闭违规内容高亮
- **导出报告**：将检测报告导出为JSON文件

---

### 右侧栏：检测报告区

#### 1. 总体评分
显示：
- **总分**（0-100）
- **风险等级**：低风险/中风险/高风险
- **统计数据**：
  - 总违规项数
  - 高风险数量
  - 中风险数量
  - 低风险数量

#### 2. 违规详情列表
每个违规项包含：
- **标题**：违规规则名称
- **位置**：第X页 · 行Y-Z
- **严重程度**：高/中/低
- **原文内容**：违规段落的内容引用
- **违反规则**：具体违反的规则（可多个）
- **修改建议**：具体可执行的建议
- **法律依据**：相关政策或规范条款

#### 3. 交互操作
- 点击违规项标题可**展开/收起**详情
- 点击违规项会自动**滚动到中间栏对应位置**并**闪烁提示**

## 🎯 使用流程

### 完整检测流程

1. **选择文档**
   - 点击左侧 "选择Markdown文档"
   - 选择要检测的 `.md` 文件

2. **预览文档**
   - 中间栏会自动显示文档内容
   - 可以先浏览文档确认内容正确

3. **开始检测**
   - 点击右上角的 **"开始检测"** 按钮
   - 等待检测完成（显示加载动画）

4. **查看结果**
   - 中间栏会高亮显示违规内容
   - 右侧栏显示详细的检测报告

5. **定位问题**
   - 点击右侧的违规项，中间栏会自动滚动到对应位置
   - 或者直接点击中间栏的高亮区域，右侧会展开对应的违规项

6. **导出报告**
   - 点击 "导出报告" 按钮
   - 保存JSON格式的完整报告

## ⚠️ 注意事项

### 1. 浏览器兼容性
- 推荐使用 **Chrome**、**Edge**、**Firefox** 最新版本
- 需要支持 ES6+ JavaScript
- 需要支持 CSS Grid 和 Flexbox

### 2. 文件路径
- 由于浏览器安全限制，需要手动选择文件
- 确保选择的文档路径下有 `images/` 文件夹（如果引用了图片）

### 3. 图片加载
- 图片路径支持相对路径
- 系统会自动在文档同级目录下的 `images/` 文件夹查找

### 4. 页码标签
- 页码标签 `<page N>` 会在渲染时自动移除
- 但在检测时会正确识别页码信息

### 5. API配置
- 确保 `.env` 文件中的API密钥正确
- DeepSeek API用于文本检测
- Qwen-VL API用于图片检测

## 🐛 常见问题

### Q1: 启动报错找不到模块？
**A**: 运行 `pip install -r requirements.txt` 安装所有依赖

### Q2: 检测一直加载中？
**A**: 检查：
- API密钥是否正确
- 网络连接是否正常
- 浏览器控制台是否有错误

### Q3: 图片无法显示？
**A**: 确保：
- 图片文件存在
- 路径正确（相对于markdown文档）
- 图片在 `images/` 文件夹下

### Q4: 高亮不显示？
**A**: 
- 点击 "切换高亮" 确保高亮已开启
- 刷新页面重新检测

### Q5: 无法导出报告？
**A**: 
- 确保已完成检测
- 浏览器是否阻止了下载

## 📊 API接口说明

### POST /api/check
检测文档合规性

**请求参数:**
```json
{
    "file_path": "教案测试2/string.md",
    "image_folder": "images"
}
```

**响应数据:**
```json
{
    "success": true,
    "original_content": "...",
    "violations": [...],
    "report": {...},
    "text_segments": [...],
    "image_segments": [...]
}
```

### GET /api/rules
获取合规规则列表

**响应数据:**
```json
{
    "success": true,
    "rules": [...]
}
```
